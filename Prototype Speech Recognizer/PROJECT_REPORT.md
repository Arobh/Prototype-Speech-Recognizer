# Project Report: Prototype Speech Recognizer

## Course

Advanced Topics in Speech Processing

## Project Title

Prototype Speech Recognizer

## Abstract

This project presents a prototype speech recognition system that converts spoken input into text. The system captures voice through a microphone and uses the Whisper automatic speech recognition model to produce text output. The project demonstrates the practical application of automatic speech recognition, including speech acquisition, waveform visualization, spectrogram visualization, MFCC feature visualization, model-based transcription, automatic model comparison, Vosk baseline comparison, processing-time measurement, noise augmentation, Word Error Rate evaluation, word-level error analysis, batch dataset testing, charts, and multilingual recognition support.

## Introduction

Speech is one of the most natural forms of human communication. Automatic speech recognition, commonly known as ASR, is the process of converting a spoken acoustic signal into written text. ASR systems are used in virtual assistants, transcription tools, accessibility systems, voice search, dictation software, call center analytics, and many other speech-based applications.

This project implements a simple speech-to-text prototype to demonstrate the basic working behavior of an ASR system.

## Aim

To develop a prototype that listens to a person's speech through a microphone and returns the recognized spoken content as text.

## Objectives

- Capture speech input from the microphone.
- Display the time-domain waveform while recording.
- Display a live spectrogram while recording.
- Display spectrogram and MFCC features for uploaded audio.
- Convert spoken words into text.
- Display final recognized text generated by Whisper.
- Measure transcription processing time.
- Compare Whisper `tiny`, `base`, and `small` models.
- Compare Whisper with a Vosk offline ASR baseline.
- Process multiple uploaded audio files in a batch.
- Apply synthetic white-noise augmentation at target SNR levels.
- Record speaker and environment metadata.
- Estimate noise level and SNR for uploaded audio.
- Evaluate transcription accuracy using Word Error Rate.
- Show inserted, deleted, and substituted words.
- Test uploaded audio samples with reference transcripts.
- Provide basic user controls for starting, stopping, clearing, and copying text.
- Demonstrate speech recognition in a simple and understandable interface.

## System Design

The system follows this flow:

```text
Speaker Voice -> Microphone Input -> Waveform and Spectrogram Display -> Whisper ASR Backend -> Text Output -> Timing and WER Evaluation
```

The application is implemented using a browser frontend and a Python backend. The browser records microphone audio using JavaScript, then sends the recording to a FastAPI backend. The backend uses Whisper through the `faster-whisper` package to perform speech-to-text conversion.

## Model Implemented

The implemented model is **Whisper ASR**.

Whisper is a deep learning speech recognition model trained for automatic speech recognition and multilingual transcription. In this project, Whisper is used through the `faster-whisper` package, which provides efficient inference using CTranslate2.

The interface supports three model sizes:

- `tiny`: fast but less accurate.
- `base`: balanced speed and accuracy.
- `small`: more accurate but slower.

The model runs locally on CPU using int8 computation.

## Modules

### 1. User Interface

The user interface is built using `index.html` and `styles.css`. It contains the project title, recording status, control buttons, language selector, Whisper model selector, waveform display, spectrogram display, recognized text display area, WER evaluator, speaker and environment metadata fields, uploaded-audio feature displays, and dataset audio test section.

### 2. Frontend Recording Logic

The recording logic is written in `app.js`. It starts and stops microphone recording using the MediaRecorder API, sends the audio recording to the backend, receives the Whisper transcript, and updates the transcript on the screen.

### 3. Waveform and Spectrogram Visualization

The waveform and spectrogram are drawn using the Web Audio API. The microphone stream is connected to an analyser node. Time-domain audio samples are rendered as a waveform, and frequency-domain energy values are rendered as a scrolling spectrogram while recording.

### 4. Whisper Backend

The backend is written in `server.py`. It receives audio through the `/transcribe` endpoint, loads the selected Whisper model, performs transcription, measures processing time, and returns the recognized text as JSON.

### 5. Transcript Handling

The system stores final results returned by Whisper. Each recording can be appended to the existing transcript, allowing the user to build a longer transcription in multiple recording segments.

### 6. Word Error Rate Evaluation

The WER module compares the expected transcript with the recognized transcript. It normalizes the text, splits it into words, calculates word-level Levenshtein distance, and reports the percentage error.

### 7. Dataset Audio Testing

The dataset test module allows the user to upload an audio file and enter its reference transcript. The file is sent to the same Whisper backend, and the output is evaluated using WER.

### 8. Feature Extraction and Noise Testing

The `/analyze-audio` backend endpoint decodes uploaded audio and computes a compact spectrogram, MFCC feature map, RMS level, estimated noise floor, and estimated signal-to-noise ratio. These values help compare clean and noisy speech samples.

### 9. Metadata Recording and CSV Export

The dataset section records speaker ID, environment condition, microphone type, and noise type. It also exports experiment rows as CSV, making the prototype suitable for small speech-processing experiments and report-ready result tables.

### 10. Automatic Model Comparison

The model-comparison mode runs the same uploaded audio through Whisper `tiny`, `base`, and `small`. This allows comparison of recognition accuracy and processing time for different model sizes.

### 11. Batch Dataset Testing

The batch testing mode accepts multiple uploaded audio files and processes each file sequentially. This supports small dataset evaluation without requiring a separate script.

### 12. Noise Augmentation

The noise augmentation mode adds synthetic white noise before transcription and feature extraction. Supported target SNR values are `20 dB`, `10 dB`, and `5 dB`.

### 13. Result Charts

The frontend draws two result charts: average WER by model and average processing time by model.

### 14. Vosk Baseline

The Vosk baseline uses a local Vosk small English model. It provides an offline ASR comparison against Whisper. This helps demonstrate how different ASR engines perform on the same sample.

### 15. Word-Level Error Analysis

The error analysis module uses word-level dynamic programming alignment. It reports substitutions, deletions, and insertions, which helps explain why a WER score is high or low.

## Algorithm

1. Load the web application in a supported browser.
2. Check whether microphone recording is available.
3. When the user clicks Start Recording, request microphone access.
4. Capture speech from the microphone.
5. Display the microphone waveform and spectrogram using the Web Audio API.
6. Store the recording as an audio blob.
7. Send the audio recording to the FastAPI backend.
8. Load the selected Whisper model.
9. Transcribe the audio using Whisper.
10. Measure Whisper processing time.
11. Display recognized text on the screen.
12. Compare recognized text with the reference transcript.
13. Calculate Word Error Rate.
14. For uploaded audio, compute spectrogram, MFCC, RMS, and estimated SNR.
15. Optionally add synthetic white noise before recognition and analysis.
16. Optionally compare `tiny`, `base`, and `small` on the same file.
17. Optionally compare Whisper against Vosk.
18. Extract substitutions, deletions, and insertions from the alignment.
19. Store speaker and environment metadata in the results table.
20. Update WER and processing-time charts.
21. Stop recording when the user clicks Stop.

## Word Error Rate Formula

Word Error Rate is calculated as:

```text
WER = (S + D + I) / N
```

Where:

- `S` = substitutions
- `D` = deletions
- `I` = insertions
- `N` = number of words in the reference transcript

This project computes the total number of word edits using Levenshtein distance.

| --- | --- | --- | --- | --- | --- | --- | --- |

## Limitations

- Accuracy depends on microphone quality and environmental noise.
- The first run requires downloading the selected Whisper model.
- Transcription is performed after the recording is stopped.
- The prototype uses a pre-trained Whisper model and does not train a custom acoustic or language model.
- The SNR estimate is approximate because it uses frame-level energy statistics rather than a separately measured noise reference.
- Noise augmentation uses synthetic white noise and may not fully represent real-world noise.
- Vosk baseline requires a downloaded Vosk acoustic model.

It captures human speech using a microphone, sends the audio to a local ASR backend, and returns the spoken content as text.
